# notExist

* Type: `notExist`

The `notExist` resolver replies the queries with an NXDOMAIN (the domain does not exist) error.

## ResolverConfigObject

No meaningful configuration is required, but a syntactically valid placeholder must be supplied. Common options:

* `{}` - empty object
* `[]` - empty array
* `""` - empty string
* `null`
* Any other syntactically valid value
